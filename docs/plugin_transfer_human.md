# 转人工插件说明文档

## 功能概述
买家发送"转人工"等关键词时，系统自动识别意图，客户端通过拼多多 API 将会话转移给指定客服。

## v2.1 更新说明

服务端 v2.1 删除了规则引擎，`transfer_human` 动作码现在只通过「意图规则」触发。

### 在意图规则中配置转人工

1. 进入 aikefu 后台 → 意图规则 → 新增规则
2. 意图标识：`transfer_human`
3. 触发关键词：`转人工,找客服,人工客服,找人工,真人客服`
4. 插件动作码：`transfer_human`
5. 立即回复话术：`好的，正在为您转接人工客服，请稍候～`

### transfer_human 处理逻辑

客户端收到 `transfer_human` 任务后：
1. 标记该买家消息为 `needs_human=True`
2. 在本地 UI 显示高亮提醒
3. 发送桌面通知
4. 上报 `/done` 表示已处理，`reply_to_buyer` 由服务端意图规则的 `auto_reply_tpl` 提供（无需客户端生成）

## 配置说明

### 指定客服（target_account）
- 填写客服的**昵称关键词**或**备注关键词**，包含匹配（不需要完整名字）
- **推荐做法**：给所有店铺的人工客服设置相同备注（如"人工客服"），然后填写备注关键词，这样多个店铺共用同一配置，无需每店单独设置
- 匹配优先级：先按关键词在昵称中查找，再在备注中查找，都命中则取第一个；未命中则回退到分配策略

### 分配策略
- `first`：始终选第一个子账号
- `least_busy`：选当前未回复消息最少的（推荐）
- `round_robin`：轮询分配
- `random`：随机选择

### 主账号过滤
系统自动过滤掉主账号（csid 不含 `cs_` 前缀），只从子账号中选择。
若指定了客服关键词则在全部账号（含主账号）中查找匹配。

## 成功判断逻辑
转移接口 `plateau/chat/move_conversation` 返回 `result.result == "ok"` 才算真正成功。
`success=true` 只代表 HTTP 调用成功，不代表业务成功。

## 常见问题

### 消息管理显示处理方式是 `-`
表示该消息被识别为 `transfer_human` 意图并触发了插件，但插件执行结果不在 AI/plugin 枚举内显示为 `-`，属正常现象。

### 转移失败 error_code=100063
"会话已经在该账号名下"：尝试把会话转给当前已持有该会话的账号，通常是选中了主账号导致。系统已自动过滤主账号，若仍出现请检查客服列表 csid 格式。

### Connection aborted 断线
正常的 HTTP keep-alive 超时，系统已内置静默重试，无需处理。

### 采集线程异常/浏览器关闭
系统已内置自动重启（最多5次，指数退避），无需手动重启。

## 已知限制
- `anti_content` 需要手动从浏览器抓包获取，过期后需重新配置
- cookies 失效后需重新登录店铺
