# 服务端 API 变更记录

本文档记录 aikefu 服务端各版本对客户端（dskehuduan）有影响的 API 变更。

---

## v2.1.0

### 变更1：规则引擎删除（对客户端无影响）

**类型**：删除  
**影响**：无

服务端已删除规则引擎模块，但客户端无需任何修改。
所有动作触发逻辑已迁移至「意图规则」（`intent_rules` 表），API 接口不变。

动作码不再与规则引擎挂钩，统一通过 `intent_rules` 表的 `action_code` 字段配置。

---

### 变更2：任务完成回复自动记录（对客户端无影响）

**类型**：服务端内部变更  
**影响**：无（客户端调用方式不变）

服务端现在会将 `/api/plugin/tasks/<id>/done` 响应中的 `reply_to_buyer` 字段
自动保存为 `direction='out'` 的消息记录，用于在服务端后台展示完整买卖双方对话（气泡视图）。

客户端只需正常调用 `/done` 接口上报结果，服务端自动处理记录。

---

### 变更3：意图规则纯文字回复路径（对客户端无影响）

**类型**：服务端新增处理路径  
**影响**：无

服务端新增了 `process_by='intent_reply'` 处理路径：
意图识别命中且无需插件时，服务端直接回复买家（无需下发插件任务）。

此路径不会产生插件任务，客户端轮询不会收到此类任务，无需任何适配。

---

### 变更4：健康检查 API（客户端可选对接）

**类型**：新增接口  
**影响**：可选（客户端可用于状态检测）

服务端新增健康检查接口：

```
GET /api/health
```

响应示例：

```json
{
  "status": "ok",
  "version": "2.1.0",
  "database": "ok",
  "redis": "ok"
}
```

客户端可调用此接口检测服务端连通性和版本信息（无需认证）。

---

### 变更5：系统设置页面（对客户端无影响）

**类型**：服务端新增页面  
**影响**：无

服务端新增 `/settings/` 配置页面，供管理员调整服务端运行参数。
客户端无需任何适配。

---

### 变更6：消息管理气泡视图（对客户端无影响）

**类型**：服务端前端变更  
**影响**：无

服务端消息管理页现在以气泡视图展示完整买卖双方对话。
客户端通过正常的 `/done` 接口上报任务完成即可，服务端会自动记录 `reply_to_buyer`。

---

## v2.0.0

- 删除 MySQL 直连，改为纯 API 模式
- 新增账号登录体系（`/api/client/login`、`/api/client/refresh`）
- 全面接入插件化架构（`/api/plugin/tasks/`）
- 新增店铺同步接口（`/api/shops/`）
