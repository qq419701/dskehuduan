## 4. 上传代码

### 方法一：通过宝塔文件管理器上传

chmod -R 755 /www/wwwroot/aikefu

## 5. 配置环境变量

在项目目录下创建 `.env` 文件：

```bash
cd /www/wwwroot/aikefu

# 创建环境变量文件
cat > .env << 'EOF'
# 系统安全密钥（必须修改为随机字符串！）
SECRET_KEY=your-very-long-random-secret-key-here-change-this

# 豆包AI配置（火山方舟）
DOUBAO_API_KEY=your-ark-api-key-here
DOUBAO_MODEL=your-doubao-model-endpoint
DOUBAO_VISION_MODEL=your-vision-model-endpoint

# 调试模式（生产环境设为False）
DEBUG=False
EOF
```

> ⚠️ **重要**：`SECRET_KEY` 必须修改为随机字符串，否则存在安全风险！

### 生成随机密钥

```bash
python3 -c "import secrets; print(secrets.token_hex(32))"
```

## 7. 配置宝塔守护进程

### 方法一：使用宝塔面板「进程守护管理器」

1. 宝塔面板 → 「软件商店」→ 搜索「进程守护管理器」→ 安装并打开
2. 点击「添加守护进程」
3. 填写以下信息：

```
名称：aikefu
启动用户：www
运行目录：/www/wwwroot/aikefu
启动命令：/www/wwwroot/aikefu/venv/bin/gunicorn -w 2 -b 0.0.0.0:6000 "app:create_app()"
进程数量：1
```

4. 点击「添加」→ 「启动」

### 方法二：使用Systemd（命令行）

创建服务文件：

```bash
cat > /etc/systemd/system/aikefu.service << 'EOF'
[Unit]
Description=爱客服AI智能客服系统
After=network.target

[Service]
Type=simple
User=www
Group=www
WorkingDirectory=/www/wwwroot/aikefu
Environment="PATH=/www/wwwroot/aikefu/venv/bin"
EnvironmentFile=/www/wwwroot/aikefu/.env
ExecStart=/www/wwwroot/aikefu/venv/bin/gunicorn \
    --workers 2 \
    --bind 0.0.0.0:6000 \
    --timeout 120 \
    --access-logfile /www/wwwroot/aikefu/logs/access.log \
    --error-logfile /www/wwwroot/aikefu/logs/error.log \
    "app:create_app()"
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
EOF

# 启用并启动服务
systemctl daemon-reload
systemctl enable aikefu
systemctl start aikefu

# 查看状态
systemctl status aikefu
```

## 8. 配置Nginx反向代理

### 在宝塔面板配置

1. 宝塔面板 → 「网站」→ 「添加站点」
2. 填写域名（如：`kefu.yourdomain.com`）
3. 点击站点名称 → 「反向代理」→ 「添加反向代理」
4. 填写：
   - 代理名称：`aikefu`  
   - 目标URL：`http://127.0.0.1:6000`

### 或手动配置Nginx

在宝塔面板 → 网站 → 站点 → 配置文件，添加：

```nginx
# 爱客服AI智能客服系统 - Nginx配置
server {
    listen 80;
    server_name kefu.yourdomain.com;
    
    # 开启gzip压缩
    gzip on;
    gzip_types text/plain text/css application/json application/javascript;
    
    # 反向代理到Flask应用
    location / {
        proxy_pass http://127.0.0.1:6000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_connect_timeout 60s;
        proxy_send_timeout 120s;
        proxy_read_timeout 120s;
    }
    
    # 静态文件直接由Nginx处理（提升性能）
    location /static/ {
        alias /www/wwwroot/aikefu/static/;
        expires 7d;
        add_header Cache-Control "public, immutable";
    }
    
    # 日志
    access_log /www/wwwlogs/aikefu.access.log;
    error_log /www/wwwlogs/aikefu.error.log;
}
```

### 配置SSL（HTTPS）

1. 宝塔面板 → 网站 → 站点 → SSL
2. 点击「Let's Encrypt」→ 申请免费SSL证书
3. 开启「强制HTTPS」

## 9. 开放端口

### 在宝塔面板开放端口

1. 宝塔面板 → 「安全」→「防火墙」
2. 点击「放行端口」
3. 端口：**6000**，备注：爱客服系统
4. 点击「确定」

### 在云服务器安全组开放端口

**阿里云/腾讯云**：

1. 进入云服务器控制台
2. 安全组 → 入站规则 → 添加规则
3. 协议：TCP，端口：6000，来源：0.0.0.0/0
4. 保存

> 如果使用Nginx反向代理（推荐），只需开放80和443端口，不需要开放6000

## 10. 初始化运行

### 启动系统

```bash
cd /www/wwwroot/aikefu

# 首次运行（自动创建数据库和默认数据）
source venv/bin/activate
python app.py
```

看到以下输出说明启动成功：
```
╔══════════════════════════════════════════════╗
║       爱客服AI智能客服系统 已启动              ║
║  访问地址：http://0.0.0.0:6000              ║
║  默认账号：admin / admin123                  ║
╚══════════════════════════════════════════════╝
```

### 首次登录配置

1. 浏览器访问：`http://服务器IP:6000`（直接访问）  
   或 `https://kefu.yourdomain.com`（Nginx代理）
2. 使用默认账号登录：`admin / admin123`
3. **立即修改密码**（点击顶部「修改密码」）

### 基础配置步骤

1. **添加行业**：系统 → 行业管理 → 添加行业
   - 设置行业代码、名称、AI提示词
   
2. **添加店铺**：店铺管理 → 添加店铺
   - 选择所属行业（同行业多店铺共享知识库）
   - 填写平台API Key
   
3. **配置AI**：修改 `.env` 文件，填写豆包API密钥
   
4. **添加知识库**：知识库 → 添加条目
   - 按行业添加问答对
   
5. **添加规则**：规则引擎 → 添加规则
   - 设置关键词触发规则

## ❓ 常见问题

### Q1: 端口6000无法访问

**A**: 检查以下几点：
1. 宝塔防火墙已开放6000端口
2. 云服务器安全组已放行6000端口  
3. 服务是否正在运行：`systemctl status aikefu`

### Q2: 提示"Internal Server Error"

**A**: 查看错误日志：
```bash
tail -50 /www/wwwroot/aikefu/logs/aikefu.log
```

### Q3: 豆包AI不回复

**A**: 检查配置：
1. `.env`文件中 `DOUBAO_API_KEY` 是否正确
2. 豆包模型endpoint是否有效
3. 测试API：在管理后台 → 知识库测试

### Q4: 数据库错误

**A**: 重新初始化数据库：
```bash
cd /www/wwwroot/aikefu
rm -f instance/aikefu.db
python -c "from app import create_app; app = create_app()"
echo "数据库已重置"
```

### Q5: 定时任务不执行

**A**: 定时任务使用北京时间，检查：
1. 服务器时区是否正确（系统时区不影响，已使用pytz）
2. 服务是否正常运行
3. 查看日志中的`[定时任务]`相关输出
